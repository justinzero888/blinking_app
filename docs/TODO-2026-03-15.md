# new features for blinking notes

- Enhancements for Habit list: There are three states for "habit lists" - 1) Available list - this is the list with items active or inactive. the frequency can be daily - such as "take vitamin", "5000 steps" etc, or once every week - for example "House cleaning every Saturday" or any pre-planned item: such as "Call AJ March 20". Or unspecified ad hoc which can be included any day manually 2) Daily list: include all active daily items plus any items that should be done that day. For example, I specify "House Cleaning every Saturday", then on March 21st, this item should be automatically showing on the daily (pending) list for that day. 3) Completed list: This is the list that "checked" for the day. Under the calendar view, each day will populate the fresh list from #2, once user check the square box, the item should be taken out from the UI, a small item icon and check box for the item should appear in a "daily habit completion area" as unchangeable record. Once the day is over, the day's habit status cannot be changed anymore. The presentation should like following (example): Vitamin: Check; 5000 steps: Cross; Sit up 20: Check; Digest: Check. Please note: each item should be represented as icon. Status is represented as Check icon and Cross (X) icon.
- Enhancements for Cards: 1) cards can be editable; 2) make the card template editable and be able to upload any picture. 3) Use AI to merge the one or more notes into one concise sentence or poem instead of copy the long original conversation. Preserve the links of original notes.
- Implement publish to social media: any notes or cards can be published to Wechat, Rednots and Telegram
- Enhancements for AI Assistant: make it personalizable: 1) can give it a name; 2) can define personality 3) make the icon more animation
