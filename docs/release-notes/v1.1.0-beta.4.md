## Changes Since v1.1.0-beta.3+18

### Bug Fixes

- **Backup date range**: Media files now filtered by selected date range (PROP-1). "Last month" backups are meaningfully smaller and faster.
- **Restore progress bar**: Non-dismissible progress dialog with percentage, time estimate, and warning during restore (PROP-2). Replaces the blank spinner.
- **Calendar routine checklist**: Simplified from 3-element to 2-element layout.
- **Template name locale**: Fixed template name field showing Chinese under English locale in edit sheet.

### Storage Hygiene (PROP-4)

- **`deleteFolder()`** now cleans up rendered PNGs for all cards in the deleted folder.
- **`deleteCard()`** now cleans up deterministic-path PNG even when `renderedImagePath` is null (edge case: preview-then-unsaved).
- **`deleteTemplate()`** now deletes the `customImagePath` image file.
- **`updateTemplate()`** now deletes old `customImagePath` when image is changed or cleared.

### Performance (PROP-5)

- **DB migration v11**: Two new indexes for faster lookups on large datasets:
  - `idx_entry_tags_entry_id` on `entry_tags(entry_id)` — speeds up entry tag resolution
  - `idx_note_card_entries_card_id` on `note_card_entries(card_id)` — speeds up card loading
- Migrations are idempotent (`CREATE INDEX IF NOT EXISTS`).

### Features

- **EntryDetailScreen**: Read-only entry detail view with Share + Post to Chorus.
- **ChorusService + PostToChorusSheet**: Social publishing to Chorus backend.
- **DB v10**: `source_template_id` on templates — enables English display names for copies of built-in templates.
